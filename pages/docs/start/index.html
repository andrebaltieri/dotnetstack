O que preciso saber? antes de começar

Esta nova versão do .NET ainda está em fase BETA, ou seja, muitas mudanças ainda estão por vir, muitos bugs ainda podem ocorrer e principalmente, recomendamos que você não coloque nada em produção utilizando estas tecnologias ainda.

Leia atentamente este guia, ele vai te mostrar os principais passos na instalação das ferramentas, vai te mostrar o que mudou na nova versão e te guiar pelas demos que preparamos para que você já comece a estudar o que vem por ai no futuro.

Ferramentas necessárias antes de começar

É extremamente importante que você se familiarize com ferramentas como Terminal (Linux/Mac) ou Command (Windows), NodeJS e NPM, GIT/GitHub, Yeoman, Bower, Grunt, Sublime, Brackets ou Visual Studio Code.

  • Terminal (Linux/Mac) ou Command (Windows)
    A famosa "janea preta" do Windows ou o Terminal no Mac/Linux. Esta vai ser a sua porta de entrada aqui, afinal somos developers, escrever códigos e comandos está no nosso sangue! Dê adeus a tudo via interface gráfica!
  • NodeJS e NPM
    Não é novidade que o Node/NPM invadiram nosso mundo, então prepare-se pois vamos utilizá-los. Sua instalação é simples e também será abordada aqui. O NPM é o gerenciador de pacotes do Node, por ele vamos baixar e intalar o Yeoman, Bower, Grunt e outros pacotes para tornar nossa vída mais fácil.
  • GIT/GitHub
    O GitHub é o currículo de um bom desenvolvedor, se você ainda não começou a utilizá-lo, comece! Este site e todas as demos estão disponíveis aqui no GitHub!
  • Yeoman
    Yeoman é um gerador de aplicações. Vamos utilizá-lo para não precisar iniciar uma aplicação do zero. Ele possui vários templates, para várias tecnologias, e entre eles, um para o novo ASP.NET!
  • Bower
    O Bower é um gerenciador de pacotes para front-end. Ele funciona como o NuGet no ASP.NET, porém só gerencia pacotes para nossa interface.
  • Grunt
    O Grunt é um gerenciador de tarefas. Com ele podemos minificar nosso CSS, fazer um pacote dos nosso JavaScript e uma outra infinidade de coisas. Este pacote vai nos auxiliar na automação das nossas atividades comuns.
  • Sublime, Brackets ou Code
    Por fim, precisamos de um editor de código de alto nível, e para nossa sorte temos três bons no mercado. O consolidado Sublime, o novato Brackets e o recém chegado Visual Studio Code!

Deu erro, e agora? antes de começar

Assim como vários outros produtos e tecnologias Microsoft, o ASP.NET tem seu código aberto e publicado no GitHub, o que significa que temos contato diretor com o time de desenvolvimento da ferramenta e podemos criar bugs, dar sugestões e até mesmo alterar o código dele e submeter esta mudança.

Porém, em muitos casos, o que ocorre é que dado um determinado erro, o desenvolvedor se desespera e sai abrindo bugs direto no GitHub. Isto é muito ruim, por que para o time de desenvolvimento para analizar um bug, que no final era algum erro de configuração do desenvolvedor.

Sendo assim, pedimos encarecidamente que quando tiver algum problema, sempre busque no Google antes, depois recorra aos fóruns e grupos do Facebook, depois a seus amigos e só então, se tiver certeza que é um bug, e desconhecido, abra um chamado para o time do ASP.NET!

Além de ajudar o time, você está aprendendo a caminhar com as próprias pernas!

Ambiente de Trabalho antes de começar

Esta nova versão do .NET pode ser configurada em qualquer ambiente Windows, Mac e Linux. Nos ambientes não-Windows você precisará de uma versão do Mono na qual também auxiliaremos na instalação. Não temos como cobrir todas as distribuições de Linux, então focaremos na mais famosa, o Ubuntu.
